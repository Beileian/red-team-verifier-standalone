# 书籍蒸馏评审方法论

> 2026-07-04 建立。来源：《买在起涨点》蒸馏评审 + 衍复投资顾王琴访谈对比。

## 两类知识的蒸馏门槛

| 类型 | 定义 | 蒸馏前提 | 示例 |
|------|------|------|------|
| **技法类** | 需要具体参数、规则、触发条件的操作性知识 | **必须获取原文**（划线/笔记/书评摘录）。仅凭TOC+简介蒸馏=空壳框架 | 起涨点识别参数、布林带斜率阈值、止损设置规则 |
| **理念类** | 投资哲学、评估框架、思维方式的认知性知识 | 访谈/书评/简介可提取核心观点 | 长期持有理念、概率管理思维、三标准评估框架 |

**判断标准**：蒸馏后的产物能否独立执行？能→理念类，做到位了。不能→技法类，缺原文。

## API 边界

- `weread.qq.com/web/book/bestbookmarks?bookId=X` — 只返回**章节统计**（哪些章有热门划线、totalCount），不返回划线文字
- 所有内容获取端点（`underlines`/`bookmarklist`/`chapter`）统一返回 `-2010 用户不存在` 或 404
- 不带 Cookie 也能拿章节分布 → 这是公开钩子，内容是收费墙

**结论**：微信读书公开API不可用于获取书内原文。蒸馏必须依赖用户自己的阅读+划线，或网上已有的读书笔记/书评。

## 蒸馏前强制检查

### 1. 框架重叠分析

新建 skill 前，必须与现有框架做逐项对比：

| 新框架 | 二姐 | 慕云 | Serenity | 真增量？ |
|--------|------|------|----------|:---:|
| （逐项填写） | | | | |

仅保留真增量作为 skill 内容。重复的部分注明"已有覆盖"。

### 2. 不入库不可建 skill

没有原文的蒸馏 → 降级为轻量对照清单（prompt 追加），不建独立 skill。

### 3. 定位优先于制作

建 skill 前先回答：
- 这个 skill 解决的是**入场**问题还是**出场**问题？
- 现有框架在哪一端已拥挤？（入场信号层：缠论+趋势+Serenity+二姐+慕云 → 已拥挤）
- 真增量在哪一端？（出场纪律层 → 当前薄弱）

## 降级策略

当蒸馏缺乏原文支撑时：

| 产物 | 适用场景 | 落地方式 |
|------|------|------|
| 轻量对照清单 | 框架理解到位，缺乏执行细节 | prompt 追加：分析时附加检查表 |
| 完整 skill | 有足够原文+量化参数 | 独立 skill + 注入三合一 |

**翻车案例**：《买在起涨点》蒸馏评分 4.4/10 🟠 C — 基于TOC+简介的五条融合建议全部是概念映射，无一句原文引用。红队指出"空壳框架"、"融合建议是推测而非验证"、"与二姐/慕云重复建设"三个致命疑点。降级为不建skill。

## 成功案例

衍复投资顾王琴访谈：一篇文章=理念类知识。核心观点（量化不预测涨跌、三标准评估框架、四条避坑法则）可直接提取并引发系统性反思，不需要逐字蒸馏。
