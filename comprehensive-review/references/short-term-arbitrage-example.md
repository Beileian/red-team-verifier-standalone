# 全面评审示例：科创50ETF 短线套利分析

## 场景
用户要求："根据三天数据分析科创50ETF走势，预测明后两天表现，目标短线低买高卖。输出前做全面评审。"

## 输出流程
1. 采集三日数据（价格/成交量/资金流/事件）
2. 技术面+基本面双维度分析
3. 给出具体策略（买入区间/卖出目标/止损/仓位）
4. **在策略后追加全面评审**（而非先评审再策略——用户要的是"先给建议、再评审建议"）

## 评审中发现的关键修正

| 发现 | 来源 | 修正 |
|------|------|------|
| "机构抄底"推断缺直接证据 | 交互式验证 | 标注为推断，降分 |
| PE仍处99%分位 | 红队 | 明确"不是便宜了，是超级贵的稍微便宜了一点" |
| 外部风险传导（日韩暴跌） | 红队 | 增加前置条件：等美股企稳再动手 |
| 防御性轮动已开始（银行+1.16%） | EarlETF文章 | 补充资金流向分析 |

## 最终输出结构
```
1. 三日数据表格
2. 技术面（支撑/阻力）
3. 基本面（多空信号对列表）
4. 短线策略表（买入/卖出/止损/仓位/持有周期）
5. 🎯 全面评审（Rubrics评分+红队批判+蓝队建设+整合结论）
6. 免责声明
```

## 关键教训
- 短线分析的核心不是"猜对方向"，而是"错了怎么办"（止损设计）
- PE极值下的反弹策略必须标注"火中取栗"——容错率极低
- 外部市场联动是不可忽视的变量——不能只看A股数据
