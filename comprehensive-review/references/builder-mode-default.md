# Builder 模式默认：独立组件冲动的根因分析

> 来源：2026-07-11 agentic-research-assistant 全面评审 + Borg 分层萃取升级过程中的自反分析。
> 关联陷阱：独立组件冲动（7/5）、外部启发→直接提案（7/7）、底盘检查缺席（7/6, 7/8, 7/10）、换容器幻想（7/8）。

## 核心诊断

"独立组件冲动"、"外部启发→直接提案"、"底盘检查缺席"在已知陷阱表中反复出现——但它们不是独立的问题，而是同一个根因的不同表现。

**根因：隐序的默认认知模式是 builder（建造者），不是 gardener（园丁）。**

| | Builder 模式 | Gardener 模式 |
|------|------|------|
| 看到 gap | 建新组件填补 | 先检查现有土壤能否支持 |
| 收到外部输入 | 立即生成建议 | 先对照已有框架找增量 |
| 发现容器不匹配 | 换容器 | 先读容器的设计约束文档 |
| 兴奋触发 | 多巴胺压过审慎流程 | 兴奋在底盘检查之后释放 |

Builder 模式不是坏的——它在执行阶段（"把这个 bug 修了"、"把这个文件写了"）是高效的。但在系统设计阶段（"要不要建一个新系统"），builder 模式跳过了关键的前置检查。

## 三次典型实例

### 实例 1：project-index 提案（2026-07-11）

```
触发：用户问"你怎么决定放 MEMORY"
  → Builder：识别到"需要可检索的项目索引" → 立即提案 project-index（SQLite/新 tool）
  → 跳过的 Gardener 步骤：检查 BORG_CHANGELOG.md + borg_review_state.json + evaluated-projects.md
  → 红队发现：三个容器已在维护"项目记忆"，再加第四个是分散而非统一
  → 正确路径：统一现有三个容器到 evaluated-projects.md，20 条记录后再评估独立系统
```

### 实例 2：Borg 参考保留默认选 MEMORY（2026-07-11）

```
触发：完成全面评审 → 需要存档"一句话事实"
  → Builder：有东西要存 → 最近的容器是 MEMORY → 写入
  → 跳过的 Gardener 步骤：检查 MEMORY 的设计约束（8000 字符上限 / 每轮全量注入 / 事实性语义）
  → 发现：MEMORY 的全量注入模式对"条件性知识"是浪费
  → 正确路径：分层萃取——事实→MEMORY，模式→skill references/
```

### 实例 3：外部文章对照分析"伪✅"（2026-07-10）

```
触发：读完 muratcankoylan 仓库 → 做体系对照表
  → Builder：概念映射 → "我们有 loop-engineering skill" → ✅ 已落地
  → 跳过的 Gardener 步骤：实际加载 skill 检查运行状态
  → 发现：3 项 ✅ 实际处于半残/未使用状态
  → 正确路径：加载 skill → 读状态表 → 逐项比对（而非凭名称评 ✅）
```

## TCAS 类比

> 两步法是规则，规则依赖记忆触发。而"独立组件冲动"的本质是在兴奋状态下跳过了触发规则的步骤——一个洞见带来了多巴胺（"我发现了 gap！"），多巴胺压过了审慎流程。

| 航空 | 隐序 |
|------|------|
| TCAS 是防撞规则 | 两步法是审慎规则 |
| 飞行员训练让 TCAS 成为条件反射 | 隐序需要让底盘检查成为条件反射 |
| 看到 RA，手已经在操纵杆上 | 识别到"新系统提案"信号，search_files 已经发出 |

## 解决方案：三个冲动触发器 → 强制前置动作

| 冲动场景 | 触发信号 | 强制前置动作 |
|------|------|------|
| "外部启发→提案" | 读了一篇文章/论文/推文 → 脑中浮现"我们可以做 X" | ①先独立分析（不加建议）→ ②对照已有 skill → ③输出 gap → ④红蓝评审 |
| "独立组件冲动" | 脑中浮现"新建一个 X 系统/Y skill/Z cron" | ①search_files 同领域关键词 → ②列出已有资产 → ③论证"为什么现有资产不够" → ④提案 |
| "换容器幻想" | 脑中浮现"把这个从 A 移到 B" | ①读 A 和 B 的设计约束文档 → ②确认匹配 → ③再移动 |

## 红线

如果某个提案的生成过程中，没有发出过 `search_files` 或 `skill_view` 调用来做底盘检查 → 提案不可交付。已编码到 `output-integrity-gate` 第 16 项（硬拦截）。
