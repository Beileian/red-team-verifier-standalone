# post_extreme 规则红队评审实例 (2026-07-13)

> 来源：科创50ETF 盘后复盘 → 两 P0 提案 → 红蓝评审 → 修正落地

## 背景

7/10(-5.36%) → 7/13(-2.78%) 连续暴跌。盘后复盘识别出四个 lessons，其中两个 P0：

- **P0-1**：post_extreme 规则 — 前一日 |涨跌|>5% → 强制 bearish/bullish skew + ATR 区间下移 20%，持续 2 个交易日
- **P0-2**：stale_SOX 回退 — SOX >24h 未刷新 → confidence 上限 low，所有 SOX gate 视为已触发

## 红队发现的补丁思维特征

红队批判识别出两个提案都是**针对最近两次失败场景的精确补丁**，具有三类典型补丁症状：

### 1. 悬崖边界（Cliff Edge）

| 症状 | 表现 |
|------|------|
| 离散魔法数字 | 5% 阈值——4.8%+3% 累积 7.6% 不触发 |
| 硬编码持续时间 | "2 个交易日"假设市场记忆只有 2 天——但第 3 天继续跌怎么办？ |
| 补丁间交互盲区 | post_extreme 覆盖第一跳的次日，覆盖不了第二跳——而这次恰好是两跳 |

### 2. 自相矛盾

| 症状 | 表现 |
|------|------|
| 参数组合互斥 | "强制 bearish_skew + ATR 下移 20%"——暴跌后 ATR 已暴涨，下移 20% 让区间窄到无法操作 |
| 补丁覆盖原有逻辑 | 强制方向偏置会覆盖技术指标的独立判断——若次日 V 型反弹（6/24 科创50 +4.2%），补丁导致错失反转 |

### 3. 过度反应

| 症状 | 表现 |
|------|------|
| 数据缺失 = 风险确认 | "所有 SOX gate 视为已触发"——SOX 停更可能是周末维护，不代表隔夜有风险 |
| 弱相关信号放大 | 科创50 与 SOX 相关系数 ~0.3-0.4，为了弱相关信号的数据缺失拉满所有门禁 |
| 系统停摆风险 | 若 SOX 停更一周，confidence 永久 low → 系统实质瘫痪 |

## 红队重定向 → 系统性修正

| 补丁 | 修正 | 为什么更系统 |
|------|------|------|
| 5% 离散阈值 | 滑动窗口：前 3 个交易日累积 > 8% 触发 | 覆盖 4.8%+3% 不触发的 cliff edge |
| 强制方向偏置 | 仅收紧入场条件（区间宽度 × 0.7），不强制方向 | 方向仍由技术指标判定，极端行情只提高门槛 |
| 2 天硬编码 | ATR 回落至暴跌前 1.3× 以内自动解除 | 匹配市场实际——波动率聚类平均持续 5-8 天 |
| SOX stale = gate 触发 | 两档：仅 SOX stale → 降档不触发；SOX+macro 双 stale → 触发 | 单一数据源缺失不瘫痪系统 |

## 与已有陷阱的关系

本案例是 `comprehensive-review` 已知陷阱第五条（2026-07-05「补丁思维 vs 设计思维」）的再次确认。核心模式不变：

> 发现失败 → 针对失败场景打精确补丁 → 补丁互相覆盖/矛盾的边界条件指数增长 → 系统比修复前更不可预测

新增教训：**盘后复盘识别的 lessons 必须过红蓝评审再落地**——复盘是诊断（找出问题），红蓝是处方审核（确定治疗方案是否对症而非头痛医头）。
