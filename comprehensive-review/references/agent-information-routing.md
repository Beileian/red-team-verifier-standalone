# Agent 驱动的财经信息源路由（v1.0.0 · 2026-07-15）

> 来源：格隆/巴伦两个财经公众号接入 IMA 知识库后的「怎么让分析工具用上这些文章」设计讨论。

## 问题

财经公众号文章自动入库到 IMA「各类公众号/」子文件夹后，分析工具（Serenity、三合一、EarlETF 等）默认只在「投资理财/」文件夹搜索研报和年报。两条路径隔开，新增信息源在分析时未被利用。

## 三种方案对比

| 方案 | 做法 | 改动成本 | API 消耗 | 噪音风险 |
|------|------|:--:|:--:|:--:|
| **被动** | 不做任何改动，agent 分析时手动 search_knowledge | 零 | 零（仅手动触发） | 低（人判断） |
| **主动** | 在分析 skill prompt 中写死降级搜索规则 | 高（每个 skill 都改） | 每次分析 +2 次 | 高（关键词匹配） |
| **清单路由** ✅ | MEMORY 存入信息源清单，agent 自主按需选择 | 极低（一行 MEMORY） | 按需（agent 判断） | 低（上下文驱动） |

## 清单路由模式

在 MEMORY 中声明可用信息源及其定位，agent 在分析时根据上下文自主判断该搜哪个源：

```
财经分析可用信息源（agent自主按需选择搜索范围）：
- IMA/投资理财/ (研报、年报、盘前审查存档)
- IMA/各类公众号/EarlETF/ (量化视角)
- IMA/各类公众号/慕云思辩/ (宏观叙事)
- IMA/各类公众号/二小姐笔记/ (板块轮动/AI硬科技)
- IMA/各类公众号/格隆/ (资本市场解读)
- IMA/各类公众号/巴伦/ (美股/港股宏观)
```

## 为什么清单路由优于主动降级搜索

1. **避免噪音**：IMA search_knowledge 是关键词匹配，没有语义排序。搜索"科创50"在格隆的文章中可能命中"韩国钢铁"而非真正相关的半导体分析
2. **避免配额浪费**：IMA 搜索日配额约 100 次，每个分析自动搜 2 个公众号 = +20% 日常消耗
3. **避免耦合**：分析 skill 不应绑定特定信息源——信息源会增减，skill 应保持通用
4. **agent 上下文判断**：分析科创 50 时 agent 知道自己在看半导体映射，自然会去搜巴伦的韩国半导体宏观文章；分析银行板块时自然不会去搜格隆的半导体 IPO 文章

## 落地方式

不仅限于财经分析——任何需要 agent 自主决定信息检索范围的任务，都可以用这个模式。核心是：**把「去哪找信息」从 agent 隐式常识变成显式 MEMORY 声明，由 agent 按上下文驱动搜索。**
