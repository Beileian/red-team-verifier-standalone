# output-integrity-gate 决策记录

## v1.12.0 — 2026-07-13

### 新增规则 #20：「距支撑仅X%」持有建议脆弱性

**触发场景**：同日科创50和机器人ETF午盘分析中，均以"距支撑仅X%"作为"持股不割肉"的主要理由。

**根因**：入场者和持仓者对同一价位区域做方向相反的判断——前者说"这里危险别进"，后者说"这里安全别走"——但分析从未解释为什么两组决策使用不同的风险框架。

**两关检查**：
- 检查A：X% / ATR% ≥ 0.5？若≥，空头趋势中不到1个交易日即可走完，"仅"字不适用
- 检查B：当前是否有MACD死叉+均线空头？若有，支撑失效概率被抬升

**关联发现**：与7/8区间端点虚构事件同源——"距下沿仅X%"的推理模式依赖两个脆弱假设：①支撑位本身精确（此前已翻车），②价格到了支撑会止跌（MACD死叉中不可靠）。

**落地**：硬拦截。必须附带明确止损价位 + 最大回撤百分比 + 与空头信号的矛盾说明。
